5
Commands — what you can type

Just type these in Claude (Cowork). No slashes or special syntax needed.

extract email

Download new emails — searches Gmail for each active supplier in your config, finds matching emails, downloads attachments to your Inbox folder. Already-downloaded emails are automatically skipped (dedup by Gmail Message ID).

extract card

Process credit card statements — reads all files in Inbox where the supplier name ends in cc, extracts card data (card last 4, statement date, due date, min due, total due), writes to DATACARD tab, moves files to cardextracted\.

extract invoice

Process regular invoices — reads all files in Inbox where supplier does not end in cc, extracts invoice fields (invoice number, date, amounts, tax), writes to DATA tab, moves files to Recorded\.

extract all

Run both — runs extract invoice then extract card in sequence. Useful after a batch email download.

remove password

Strip PDF passwords from Inbox — opens each file using the password in your config and saves it without password protection. Files stay in the same folder. Default target: Inbox.

remove password in cardextracted

Strip passwords from cardextracted folder — same as above but targets a different folder. You can also use remove password in recorded or remove password in error.

💡 Typical daily workflow: extract email → extract all → done. Review results in Invoice_Logs.xlsx.
6
How routing works — CC vs regular invoices

Claude looks at the last 2 characters of the Supplier/Vendor name to decide which tab to write data to:

💳 Credit card statement

Supplier name ends in cc (case-insensitive)
e.g. RAK CC, EI CC, ADCB CC

→ Data written to DATACARD tab
→ File moved to cardextracted\

🧾 Regular invoice

Supplier name does not end in cc
e.g. Emirates Paper Mart, Al Shifa Trading LLC

→ Data written to DATA tab
→ File moved to Recorded\

7
Duplicate prevention — how it works
📧

Extract email — hard skip

Each downloaded email's Gmail Message ID is saved in the DOWNLOAD_LOG. If you run extract email again on the same date range, Claude checks the log first — any email already downloaded with status SUCCESS is skipped entirely. No duplicate downloads.

📄

Extract card / invoice — soft flag

Files are never skipped (you may have added them manually to Inbox). But if a filename already appears in EXTRACTION_LOG with status SUCCESS, the extracted data row is marked _suspect duplicate in the Remarks column so you can review and delete it if needed.

Viewing your results

Open Invoice_Logs.xlsx in the root of your invoices folder. It contains 5 tabs:

DL

DOWNLOAD_LOG

Every email download attempt — supplier, date, original filename, saved filename, status, Gmail Message ID.

EX

EXTRACTION_LOG

Every file processed — outcome (SUCCESS / FAILED / SKIPPED), where it was moved, suspect duplicate flags.

DC

DATACARD

Extracted credit card data — card last 4, statement date, period, due date, minimum due, total due.

DA

DATA

Extracted regular invoice data — invoice number, date, supplier name, amounts, tax.

ER

ERROR_LOG

Files that failed — error type, suggested fix, which config column to correct.

⚠️ Do not edit Invoice_Logs.xlsx — Claude overwrites it each run. Open it to view, then close it before running any command.
10
Troubleshooting

❌ "No emails found" for a supplier

Check that From Email and Subject Keyword in your config exactly match what arrives in Gmail. Also verify that Date From / Date To covers the period the email was sent. You can test by manually searching in Gmail with those same terms.

❌ Password error when opening PDF

Double-check the password format in config column J. Common mistakes: RAKBANK uses the last 4 digits of the card (not a fixed code); ENBD uses first 4 chars of name exactly as embossed on card (no middle initials). Re-read the password hint inside the bank's email.

❌ Config not being read / Claude using old data

The config file must be in the configs\ subfolder of the connected folder. Make sure you saved and closed it in Excel before running the command. If Claude still shows stale data, try: save the file, close Excel completely, wait 5 seconds, then retry the command.

❌ Invoice_Logs.xlsx looks blank or broken

Close the file in Excel completely (including from the taskbar) and tell Claude to "re-copy the log file". Claude keeps a working copy in memory and will push it back to the folder.

❌ Chrome extension not connecting

Make sure the Claude in Chrome extension is installed and enabled in Chrome extensions page (chrome://extensions). The extension icon should be visible in your Chrome toolbar. If Claude says "multiple browsers connected", click the extension icon and choose "Connect" in the correct browser window.

❌ File ends up in Error\ folder

Open Invoice_Logs.xlsx → ERROR_LOG tab. The Suggested Fix and Config Column to Fix columns tell you exactly what to correct. After fixing the config, move the file back to Inbox manually and run extract card or extract invoice again.
